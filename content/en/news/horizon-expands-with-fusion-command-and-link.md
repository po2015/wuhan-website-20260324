Following the public introduction of Horizon Sense, Cyrentis announced the next visible set of Horizon platform components in May 2025, extending the software direction from device management into operational workflow.

The newly announced module set includes **Horizon Fusion**, **Horizon Command**, **Horizon Archive**, and **Horizon Link**. Together, these components are intended to move the software beyond simple device access and toward a unified operating environment where target correlation, alarm handling, evidence continuity, and external integration are managed as one connected system.

Horizon Fusion is positioned to correlate outputs from multiple sensing layers and reduce duplicated alarms. Horizon Command is intended to provide the main operator workspace for live monitoring and response coordination. Horizon Archive focuses on playback, event review, and retained operational memory. Horizon Link is designed as the external integration edge for third-party systems, site infrastructure, and future interface requirements.

Cyrentis stated that this release direction reflects how monitoring systems are actually used in practice. Operators do not need isolated features delivered one by one; they need software that can preserve context from sensing, to understanding, to response, and then into archive and external systems without forcing repeated manual interpretation at every boundary.

The current public structure of these modules can be reviewed through the [Horizon page](/horizon/), where the platform is presented as one connected software story rather than a set of disconnected tools.
