# Taxonomy Guidance

## Rule

For this site, `categories` should stay broad and navigational. `tags` should stay narrower and topic-specific.

Do not repeat the same term in both fields.

## Good Category Examples

- `Radar Planning`
- `Radar Architecture`
- `System Design`
- `Frequency Selection`
- `Technology Basics`
- `Deployment`
- `Compliance`
- `Export Control`
- `Digital RF`
- `Emerging Radar`
- `Emerging Sensors`
- `Machine Vision`
- `Counter-UAS`
- `Civil Applications`
- `Manufacturing`
- `Technology Watch`

## Good Tag Examples

- `C Band`
- `X Band`
- `Ku Band`
- `Phased Array`
- `AESA`
- `Mechanical Scan`
- `Electronic Scan`
- `Direct RF Sampling`
- `Digital Beamforming`
- `SAR`
- `FMCW LiDAR`
- `Layered Surveillance`
- `Dual-Use Review`
- `Export Licensing`

## Selection Rule

- Use `1-2` categories
- Use `2-4` tags
- Categories should help users browse a subject area
- Tags should help users find a narrower topic or method
