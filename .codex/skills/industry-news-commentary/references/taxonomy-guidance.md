# Taxonomy Guidance

## Rule

For news posts, `categories` should remain broad and section-like. `tags` should capture the specific topic, event type, or implication.

Do not repeat the same term in both fields.

## Good Category Examples

- `Company News`
- `Platform Updates`
- `Business News`
- `Corporate Message`
- `Industry View`
- `Electro-Optical`
- `Horizon`

## Good Tag Examples

- `Corporate Launch`
- `Contract Signing`
- `Software Roadmap`
- `Fusion Module`
- `Counter-UAS`
- `Airport Security`
- `Export Policy`
- `Low-Altitude Monitoring`
- `Project Delivery`

## Selection Rule

- Use `1-2` categories
- Use `2-4` tags
- Categories should help readers scan the type of news
- Tags should capture the specific news angle
