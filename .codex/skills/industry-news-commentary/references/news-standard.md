# News Standard

## Goal

Turn a current external event into a useful Cyrentis news article that improves site freshness, topical relevance, and search visibility without becoming empty commentary.

## Best Use Cases

- security regulation updates
- airport, port, or industrial monitoring policy changes
- counter-UAS or low-altitude awareness developments
- radar, EO, RF, or command-software launches by major players
- significant procurement, contract, or integration trends

## Required Questions

Every article should answer:

1. What happened?
2. When did it happen? This is primarily for source control and factual accuracy, not for date-heavy wording in the published body.
3. Why should a security or sensing audience care?
4. What does it suggest about system design, procurement, or operations?
5. Where does Cyrentis have a relevant perspective?

## Commentary Rule

The Cyrentis viewpoint should be explicit but disciplined.

Allowed:

- planning implications
- integration implications
- workflow implications
- product-category relevance
- likely opportunity areas

Not allowed:

- unsupported self-praise
- invented customer relevance
- exaggerated market claims
- pretending an external event is direct proof of Cyrentis capability

## Suggested Structure

1. Opening paragraph: event summary
2. Context paragraph: why the event matters
3. Analysis section: implications for security, sensing, or deployment
4. Cyrentis view: where the event intersects with Horizon, sensors, or services
5. Short closing paragraph
6. `Reference Reading`: list the authoritative source first, then add other useful external links for deeper reading

## Archive Wording Rule

- Do not write `recent`, `recently`, `today`, `this week`, `currently`, or similar relative-time phrasing in the article body
- Avoid leading with explicit dates unless the date itself changes the meaning
- Prefer durable wording such as:
  - `A new policy signal suggests...`
  - `A newly announced program indicates...`
  - `An industry move now points toward...`
- The article should still be factually anchored, but it should read cleanly when revisited months later

## Source Rule

- The main body should rely on one authoritative source
- If an inline source is used, keep it to one external link in the body
- Add other relevant external links under `Reference Reading`
- Prefer official announcements, regulators, listed-company filings, reputable trade press, or major research organizations
- `Reference Reading` can include multiple external links if each one adds distinct value

## SEO Rule

- Title should include the event topic, not just "Cyrentis comments on..."
- Use keywords naturally in title, slug, SEO title, and intro
- Keep the article specific without relying on temporary wording such as `latest` or `recent`
